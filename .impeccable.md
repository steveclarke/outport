## Design Context

### Brand Personality
**Reliable, clean, smart.** Trustworthy infrastructure that just works — smart defaults, no drama. The tool is invisible when everything's fine and helpful when something needs attention.

### Aesthetic Direction
- **Tone:** Polished product — feels like a real SaaS dashboard, not a developer utility dump. Something you'd show to someone.
- **Reference:** Docker Desktop containers view — clean list layout, status indicators, clear hierarchy, full-width usage.
- **Anti-references:** Generic AI dashboards with cards floating in whitespace. Terminal-aesthetic monospace everything. Dark mode with neon accents.
- **Theme:** Light and dark mode.

### Brand Identity
- **Primary color (navy):** `#031C54` — headings, logo primary paths
- **Accent (steel blue):** `#2E86AB` — links, buttons, interactive elements
- **Background (warm cream):** `#faf8f5` — page background
- **Soft background:** `#f5f0e8` — section backgrounds, group headers
- **Surface:** `#ffffff` — cards, elevated content
- **Text:** `#1e293b` (primary), `#475569` (secondary), `#64748b` (muted)
- **Heading font:** Barlow Bold — tight letter-spacing (-0.01em to -0.02em)
- **Body font:** Inter — variable weight
- **Mono:** SF Mono, Fira Code
- **Logo:** SVG mark (navy + steel blue) with horizontal and stacked variants at `brand/svg/`

### Design Principles
1. **URL-first** — Web service URLs are the primary content. Everything else is supporting detail. If you can't find and click the URL you need in under 2 seconds, the design has failed.
2. **Full-width, no waste** — The dashboard fills the viewport. No centered narrow columns, no cards floating in empty space. Every pixel should serve the content.
3. **On-brand** — Use the established Outport palette (navy, steel blue, warm cream), fonts (Barlow headings, Inter body), and visual language. The dashboard should feel like part of the same product as the docs site.
4. **Progressive disclosure** — Show what matters (URLs, status), collapse what doesn't (infra services, worktree instances). Expandable on demand.
5. **Polished, not utilitarian** — This should feel like a product someone built with care, not a debug panel. Clean typography, intentional spacing, branded touches.
